## Paths de uso ejecutados

### PATH A — Operador diario (kiosco + caja)

```
Login → (auto sucursal) → Abrir caja → Kiosco
  → Elegir producto → Modificadores → Agregar al carrito
  → [Editar línea | Eliminar línea | Notas | Cantidad +/-]
  → Capturar pagos (efectivo / tarjeta / transferencia / mixto)
  → COBRAR → Carrito vacío → Pedido en panel activos
  → Preparar → Entregar (o Cancelar)
  → Cerrar caja → Conteo físico → Nueva apertura
```

**Resultado:** flujo principal **funcional**. Validaciones de pago insuficiente, carrito vacío y límites de cantidad **OK**.

### PATH B — Admin back-office

```
Login admin → Panel /admin → [Sucursales | Usuarios | Catálogo | Ingredientes | Inventario]
  → Reportes (filtro fechas + CSV)
  → Gastos (puntual + recurrente)
  → Finanzas (P&L + rentabilidad por producto)
  → Ajustes (nombre, retícula modificadores)
```

**Resultado:** rutas cargan sin error 500. Gastos, finanzas, reportes y ajustes (retícula) **OK**. CRUD sucursales/usuarios/ingredientes **no confirmado por UI** (ver errores).

### PATH C — Seguridad y permisos

```
Sin sesión → /kiosk, /admin, /cash → /login
Login inválido → mensaje genérico
Logout → sesión destruida
OPERATOR → /admin redirige a /kiosk (test auditoría falló en paso crear usuario; ver E-03)
```

**Resultado:** guards de sesión **OK**. Permisos OPERATOR **no verificados end-to-end** en esta corrida.

### PATH D — Patrones destructivos / edge cases

| Patrón | Acción | Resultado |
| --- | --- | --- |
| Carrito vacío | COBRAR | Botón deshabilitado OK |
| Sin modificador obligatorio | Agregar | Deshabilitado + error rojo OK |
| Pago < total | COBRAR | `alert()` bloquea envío OK |
| Sin pagos | COBRAR | `alert()` OK |
| Doble submit | COBRAR rápido | 1 sola orden OK |
| Qty mínimo | Menos × N | Se queda en 1 OK |
| Qty máximo | Más × 105 | Tope 99 OK |
| Editar + cancelar | Oreo agregado luego cancelar | Total sin cambio OK |
| Caja cerrada | ir a /kiosk | Redirect /cash/open OK |

## Errores y hallazgos documentados

### E-01 — Duplicidad de enlaces en panel admin (test, no bug de app)
- **Dónde:** `/admin`
- **Qué:** `getByRole('link', { name: 'Reportes' })` resuelve 2 elementos (sidebar + card del panel).
- **Impacto:** Ninguno para usuario; dificulta automatización estricta.
- **Severidad:** Baja (test)

### E-02 — Crear/editar sucursal no verificado en UI
- **Test:** `full-audit` sucursales
- **Síntoma:** `getByText('Sucursal E2E …')` timeout 10s.
- **Hipótesis:** El listado muestra el nombre dentro de `<input defaultValue="…">`, no como texto visible; `getByText` no encuentra valores de input.
- **Acción pendiente:** Verificar en DB si la sucursal se creó; corregir aserción a `toHaveValue` (fuera de alcance ahora).
- **Severidad:** Media (cobertura inconclusa)

### E-03 — Crear operador + prueba de permisos inconclusa
- **Test:** `full-audit` usuarios OPERATOR
- **Síntoma:** Email del operador no visible tras Crear.
- **Hipótesis:** Mismo patrón input vs getByText; o falta `fill` del campo nombre en formulario (solo se llenó email en selector ambiguo).
- **Severidad:** Media (permisos OPERATOR no confirmados en esta corrida)

### E-04 — Tests buscan etiqueta "Pagado" (estado viejo)
- **Tests:** `kiosk.spec.ts` líneas 68, 81, 241
- **App actual:** Estado `PENDING` se muestra como **"Pendiente"** (`kiosk-client.tsx`).
- **Impacto:** Venta **sí funciona** (carrito vacío, orden creada); solo falla la aserción de label.
- **Severidad:** Alta para suite CI (tests desactualizados); **no es regresión de venta**

### E-05 — Test ciclo PAID → PREPARING → READY → DELIVERED obsoleto
- **Test:** `kiosk.spec.ts:216`
- **App actual:** `PENDING → PREPARING → DELIVERED` (sin READY; botones "Preparar" / "Entregar").
- **Verificado alternativo:** `full-audit` ciclo Pendiente→Preparando→Entregar **pasa**.
- **Severidad:** Alta (test drift)

### E-06 — Crear producto: creado pero assert incorrecto
- **Test:** `full-audit` catálogo
- **Evidencia:** strict mode — `Producto E2E …` aparece en 2 `<option>` (selects de grupo/receta).
- **Conclusión:** **createProductAction probablemente OK**; falta aserción en listado (`input[name=name]`).
- **Severidad:** Baja (test)

### E-07 — Crear ingrediente + inventario inconcluso
- **Test:** `full-audit` ingredientes
- **Síntoma:** `getByText('Ingrediente E2E …')` timeout.
- **Hipótesis:** Nombre solo en `<input>` del listado; movimientos no ejecutados porque falló paso previo.
- **Severidad:** Media (cobertura inconclusa)

### E-08 — Aserción incorrecta en caja con ventas
- **Test:** `full-audit` caja con ventas
- **Esperado por test:** `25.00` en página cierre
- **Observado:** `Efectivo vendido Q 50.00` (acumulado de ventas previas en la misma sesión de caja)
- **Conclusión:** La función **sí refleja ventas**; fallo por **aislamiento de tests**, no por bug de caja.
- **Severidad:** Baja (test)

### E-09 — Nombre empresa no visible en sidebar colapsado (desktop)
- **Test:** `full-audit` ajustes
- **Qué funciona:** `Guardar Ajustes` persiste — título de página/metadata = `E2E Co …` (visible en HTML de `/cash/close`).
- **Qué falla visual:** Sidebar desktop colapsado (`lg:w-[5.5rem]`) oculta texto del nombre (`opacity-0 w-0`) hasta hover.
- **Impacto UX:** En desktop sin hover, operador no ve el nombre customizado en sidebar; sí en `<title>`.
- **Severidad:** Media (UX / accesibilidad)

### E-10 — Documentación desactualizada (no runtime)
- **Dónde:** `docs/APP_CONTEXT.md` aún describe `PAID → PREPARING → READY → DELIVERED`.
- **Código/DB:** `PENDING → PREPARING → DELIVERED`.
- **Severidad:** Baja (docs)

### E-11 — Tests E2E legacy no actualizados tras migración enum
- **Relacionado:** Migración `20260609120000_update_order_status_enum`.
- **Síntoma histórico:** Runtime error `invalid input value for enum "OrderStatus": "PENDING"` si DB sin migrar.
- **Estado en corrida:** DB migrada; kiosco carga OK.
- **Severidad:** Resuelto en DB; tests aún con labels viejos (E-04, E-05).

### E-12 — Modificador "Crema" no encontrado (timeout 30s)
- **Test:** `kiosk.spec.ts:266`
- **Flujo:** Vaso → Solo Fresa → click `button /^Crema/`
- **Posibles causas:**
  1. Retícula táctil activada por test previo (`full-audit` ajustes) cambia estructura del botón; nombre accesible puede ser solo inicial "C" en lugar de "Crema".
  2. Botón fuera de viewport / requiere scroll.
  3. Grupo "Base" colapsado o no visible sin scroll.
- **Severidad:** Media — requiere repro manual con retícula on/off
